Infos pratiques (exemples concrets)

Pour que ce soit clair et rassurant, voici comment Sevor fonctionne dans la vraie vie : prix, dépôt, retour, litiges, bonnes pratiques.

🧾 Prix & conditions

Owner decides

Le propriétaire définit : le prix (jour/heure), la durée minimale, la disponibilité, et les règles (ex: pas de livraison, ou livraison payante).

  • Prix flexible selon la demande
  • Détails importants: état, accessoires inclus
  • Règles: permis, âge, usage pro/perso
Exemple Une caméra louée 25$/jour, minimum 2 jours, accessoires inclus (batterie + trépied).

🔒 Dépôt de garantie

Protect both

Le dépôt sert à protéger les deux parties : il est bloqué pendant la location et libéré si le retour est conforme.

  • Évite les “arnaques” et rassure
  • Encourage un retour propre
  • Réduit les conflits
Exemple Une perceuse à 10$/jour avec un dépôt de 60$. Retour OK ⇒ dépôt restitué.

📸 Photos & état

Before / after

Pour éviter les disputes, la meilleure pratique est de prendre des photos au moment du départ et du retour (même rapides).

  • Photos claires (angles + défauts)
  • Accessoires visibles
  • Heure + date si possible
Exemple “Photo du coin cassé déjà présent” ⇒ évite qu’on te le reproche après.

⏱️ Retard & prolongation

Simple rules

Si le locataire est en retard, la règle doit être annoncée dès le départ (frais ou facturation supplémentaire).

  • Prolongation possible si disponible
  • Frais de retard (ex: 1h = X$)
  • Communication rapide = moins de stress
Exemple Location d’un vélo : retard +1h ⇒ +5$ si noté dans les conditions.

⚖️ Litiges & solutions

Fair process

En cas de problème, on se base sur : conditions écrites, photos, échanges, et logique. L’objectif : rester juste.

  • Discussion d’abord
  • Preuves (photos + description)
  • Solution: réparation / compensation
Exemple Rayure “après” + non présente “avant” ⇒ prise en charge possible via dépôt (selon règles).

✅ Bonnes pratiques

Trust booster

Des détails simples augmentent énormément la confiance et les réservations.

  • Description précise (taille, modèle, compatibilité)
  • Réponse rapide aux messages
  • Propreté + rangement à la remise
Exemple “MacBook Pro M1 16GB + chargeur original” ⇒ plus de réservations qu’un texte vague.
✅ Astuce : plus une annonce est claire (photos + règles + contenu), plus elle inspire confiance ⇒ plus elle se loue.
🟣 Règles visibles 🔵 Photos avant/après 🟢 Dépôt transparent ⚪️ Communication rapide

Practical info (real examples)

To make everything crystal-clear, here’s how Sevor works in real life: pricing, deposit, returns, disputes, and best practices.

🧾 Pricing & rules

Owner decides

Owners set the price (per day/hour), minimum duration, availability, and rules (pickup only, delivery fees, requirements).

  • Flexible pricing based on demand
  • Important details: item condition + included accessories
  • Rules: ID required, age limits, pro/personal use
Example A camera rented $25/day, minimum 2 days, accessories included (battery + tripod).

🔒 Security deposit

Protect both

The deposit protects both sides: it’s held during the rental and released if the return is compliant.

  • Reduces fraud and increases trust
  • Encourages clean returns
  • Prevents unnecessary disputes
Example A drill at $10/day with a $60 deposit. Return OK ⇒ deposit refunded.

📸 Photos & item condition

Before / after

Best practice: take quick photos at pickup and return (even 20 seconds). It avoids most conflicts.

  • Clear photos (angles + existing defects)
  • Accessories visible
  • Date/time if possible
Example “Photo of an existing scratch” ⇒ prevents being blamed later.

⏱️ Late return & extension

Simple rules

Late rules should be written upfront (fees or extra billing). Extensions are easy if availability allows.

  • Extension if the calendar is free
  • Late fee example: 1 hour = $X
  • Fast communication = fewer issues
Example Bike rental: late by +1 hour ⇒ +$5 (if listed in the rules).

⚖️ Disputes & solutions

Fair process

If something goes wrong, the resolution should rely on written rules + photos + messages, with a fair outcome.

  • Start with discussion
  • Use proof (photos + listing description)
  • Fix/repair or partial compensation
Example A scratch appears “after” but not “before” ⇒ deposit can cover it (based on rules).

✅ Best practices

Trust booster

Simple details massively increase trust and bookings.

  • Precise description (model, size, compatibility)
  • Fast replies to messages
  • Clean pickup/return
Example “MacBook Pro M1 16GB + original charger” ⇒ gets more bookings than vague text.
✅ Tip: clearer listings (photos + rules + details) ⇒ more trust ⇒ more rentals.
🟣 Clear rules 🔵 Photos before/after 🟢 Transparent deposit ⚪️ Fast communication
🔎 Optional later: Terms + Help Center + Report system.
